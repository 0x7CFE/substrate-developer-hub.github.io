Front-End Developer

So you wanna build amazing user experiences...

  1. 背景

    Substrate 通过 JSON-RPC 暴露了一系列接口。为了能和这些接口交互，你的数据需要根据标准的编解码器进行编码和解码。

    为了让你的开发更容易，我们提供了一个响应式的 Javascript API，封装了所有 Polkadot 或者 Substrate 节点暴露的 RPC 方法，并且能够处理所有的编码和解码。

    根据下面的步骤，你将能够为 Substrate 和 Polkadot 编写自己的前端应用。请通过 Riot 加入我们的社区，如果你有任何问题，请随时问这些人，他们的标签为

    StackOverflowRiot Chat

  2. 使用 Polkadot-JS Apps 来了解 Substrate

    开始写代码之前，你应当熟悉 Substrate 提供了哪些功能。Polkadot-JS Apps 包含一系列应用，可以与 Substrate 的标准 runtime 模块进行交互。这些应用让你能够了解所有的内置功能，提供了非常多的额外信息，甚至能够与你自定义的 runtime 模块进行交互。

    你甚至不需要安装任何东西，只需从 Parity 搭建的 Polkadot 和 Substate 节点开始！

    Polkdot-JS Apps

  3. 深入 Polkadot-JS 生态

    Polkadot-JS Apps 是基于 Polkadot-JS API 构建的，Polkadot-JS API 为 Polkadot 和 Substrate 应用链提供了类型安全的 Javascript API。除了这些，我们还提供了一系列丰富的模块和组件让前端开发变得更加简单。所有组件都发布在 npmjs.com，并且在 Polkadot-JS 开发者门户有文档说明。

    像一个来自未来的 Substrate 应用开发者一样，随处看看，探索你的各种工具。

  6. 探索更多关于 Polkadot-JS

    如果你有兴趣看另一个 Javascript API，请探索 `oo7-bonds` 代码仓库。

    “Bonds” 是一个可选的 JSON-RPC 包装器，用来封装 Polkadot 和 Substrate的API，使用 Javascript 编写，附带自己的 SCALE 编解码实现。Substrate UI 是一个基于 React 的单页面应用，“Bonds” API 最好和 Substrate UI 一起使用。
