The ink! CLI

为了帮助简化创建 ink! 项目的过程！ 我们提供 cargo contract：一个 ink! CLI.

说明：ink! CLI 正在快速开发中，其大部分命令还没有实现！

安装

你可以使用 Cargo 安装 cargo contract 工具：

cargo install --force --git https://github.com/paritytech/ink cargo-contract

你可以用 cargo contract --help 来查看你可以使用的命令。

创建一个 ink! 项目

我们将会用到 ink! CLI 去生成 Substrate 智能合约项目所需要的文件。

Make sure you are in your working directory, and then run:

cargo contract new <name>

这个命令会创建一个指定名称的新项目文件夹，其中包含以下文件：

flipper
|
+-- .cargo
|   |
|   +-- config      <-- Compiler Configuration (Safe Math Flag)
|
+-- src
|   |
|   +-- lib.rs      <-- Contract Source Code
|
+-- build.sh        <-- Wasm Build Script
|
+-- rust-toolchain  <-- Rust Version Used
|
+-- Cargo.toml
|
+-- .gitignore

Rust 工具链

为了减少 Rust 本身 breaking change 的负面影响， 通过使用 rust-toolchain 文件，你新生成的 ink! 项目将使用特定的 nightly 版本编译器。 这意味着，你还需要确保为这个特定的编译器版本安装 wasm32-unknown-unknown，否则会出现如下错误：

error[E0463]: can't find crate for `core`
  |
  = note: the `wasm32-unknown-unknown` target may not be installed

要解决此错误，你应检查rust-toolchain文件中使用的 Rust 版本，并运行以下命令：

rustup install <version>
rustup target add wasm32-unknown-unknown --toolchain <version>

Cargo Config

你生成的 ink! 项目将包含一个 /.cargo/config 文件，用于为项目设置 rustflags 。

一个值得注意的设置是 overflow checks=on，它告诉 Rust 编译器在发生意料之外的溢出时进行异常退出处理。 这意味着合约函数自身是“安全的”，不需要补充库就可以避免所有上溢和下溢行为。

Flipper 合约模板

Ink CLI 会自动生成 "Flipper" 合约源代码 作为模板提供给你。

Flipper 合约只不过是通过 flip() 函数把一个 bool 值从 true 变到 false。 如果您想了解此合约是如何从头开始构建的，请查看创建你的第一个合约。

Wasm 构建脚本

Ink! CLI 还生成一个名为 build.sh 的构建脚本。 此文件用于帮助你将合约源代码编译为 Wasm。 这个脚本依赖于你安装的各种 Wasm 工具来设置 ink! 环境。

你只需运行：

./build.sh

如果一切顺利，你应该看到一个新创建的 target 文件夹，其中包含4个与脚本中的步骤相对应的相关文件：

target
├── flipper.wasm
├── flipper.wat
├── flipper-fixed.wat
└── flipper-pruned.wasm

最终的、优化后的 flipper-pruned.wasm 是我们将会部署到 Substrate 链上的文件。

合约ABI

如果运行了构建脚本，你应该注意到一个新生成的 JSON 文件：

Flipper.json

这是你合约应用的二进制接口 (ABI)。 让我们看看里面的内容：

{
  "name": "Flipper",
  "deploy": { "args": [] },
  "messages": [
    {
      "name": "flip",
      "selector": 970692492,
      "mutates": true,
      "args": [],
      "return_type": null
    },
    {
      "name": "get",
      "selector": 4266279973,
      "mutates": false,
      "args": [],
      "return_type": "bool"
    }

你可以看到，此文件描述了可用于与合约交互的接口。

如果在实例化新合约时需要任何部署变量，这些变量将在 deploy 部分中定义。 合约公开的所有公共函数都可以在 messages 中找到，包括函数名、函数参数、返回类型以及函数是否为只读。

还有一个 selector 是函数名的哈希，用于将合约调用路由到正确的函数。

用户界面和其他合约可以使用此文件与你的合约进行交互。
